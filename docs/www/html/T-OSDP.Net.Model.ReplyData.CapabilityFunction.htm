ContactStatusMonitoring

The compliance level of contact status monitoring.

Compliance Levels:

01 - PD monitors and reports the state of the circuit without any supervision. The PD encodes the circuit status per its default interpretation of contact state to active/inactive status.

02 - Like 1, plus: The PD accepts configuration of the encoding of the open/closed circuit status to the reported active/inactive status. (User may configure each circuit as "normally closed" or "normally open".)

03 - Like 2, plus: PD supports supervised monitoring. The operating mode for each circuit is determined by configuration settings.

04 - Like 3, plus: the PD supports custom End-Of-Line settings within the Manufacturer's guidelines.

Number Of: The number of Inputs.

OutputControl

The compliance level of output control.

Compliance Levels:

01 - The PD is able to activate and deactivate the Output per direct command from the ACU.

02 - Like 1, plus: The PD is able to accept configuration of the Output driver to set the inactive state of the Output. The typical state of an inactive Output is the state of the Output when no power is applied to the PD and the Output device (relay) is not energized. The inverted drive setting causes the PD to energize the Output during the inactive state and de- energize the Output during the active state.

03 - Like 2, plus: PD supports supervised monitoring. The operating mode for each circuit is determined by configuration settings.

04 - Like 3, plus: the PD supports custom End-Of-Line settings within the Manufacturer's guidelines.

Number Of: The number of Outputs.

CardDataFormat

The form card data format presented to the control panel.

Compliance Levels:

01 - The PD sends card data to the ACU as array of bits, not exceeding 1024 bits.

02 - The PD sends card data to the ACU as array of BCD characters, not exceeding 256 characters.

03 - The PD can send card data to the ACU as array of bits, or as an array of BCD characters.

Number Of: N/A, set to 0.

ReaderLEDControl

The compliance level of reader LEDs.

Compliance Levels:

01 - The PD support on/off control only.

02 - The PD supports timed commands.

03 - Like 2, plus bi-color LEDs.

04 - Like 2, plus tri-color LEDs.

Number Of: The number of LEDs per reader.

ReaderAudibleOutput

Compliance Levels:

Number Of: This field is ignored.
